هر ادعای گزارش پیوست با سورس‌کد واقعی پروژه مقایسه و تأیید شد. یافته‌های درست وارد گزارش نهایی شدند.

ادعای گزارش پیوست وضعیت واقعیت کد
~
SEC-01: کلیدهای پروداکشن واقعی در env نشت دارند
نیمه‌درست
.env.local حاوی Google OAuth واقعی + Stripe test key + Resend API key است. اما .gitignore درست تنظیم شده. خطر واقعی هست ولی «پروداکشن» نیست — باید revoke و migrate شود.
✕
SEC-02: NextAuth هر ایمیل را بدون بررسی تأیید می‌کند
نادرست
lib/auth.ts دارای CredentialsProvider با تأیید token از DB + بررسی expiry 15 دقیقه‌ای + single-use token است. این ادعا کاملاً غلط است. اما یک مشکل جدیدتر در middleware کشف شد (ببینید SEC-02 واقعی).
✓
DB-01: استفاده از SQLite در محیط Serverless
تأیید شد
علی‌رغم کامنت «Migrated to PostgreSQL» در schema.prisma، خط ۱۲: provider = "sqlite" — این یک مانع جدی است که باید قبل از deploy رفع شود.
✕
BUG-01: فرم رزرو تیکت را در DB ثبت نمی‌کند
نادرست
PricingConfirmation.tsx خط ۱۷۹: fetch('/api/tickets', {'{'}method:'POST'{'}'}) وجود دارد. booking به درستی ارسال می‌شود.

PERF-01: حلقه rAF مداوم در CinematicHero
نیمه‌درست
useCinematicScroll.ts خط ۱۲۱: rAF یک بار در mount بدون دلیل اجرا می‌شود. حلقه فقط هنگام اسکرول فعال می‌شود (خط ۱۰۶-۱۰۷) — نه «مداوم» اما یک waste واقعی است.

مشکلات بحرانی — رفع پیش از هر deploy

SEC-01 جدید ✓
.env.local
کلیدهای سرویس‌های خارجی در فایل محلی
فایل .env.local حاوی Google OAuth Client Secret واقعی، Stripe test key، و Resend API key است. اگرچه .gitignore درست تنظیم است، اما این فایل در RAR آپلود‌شده بود و ممکن است در سیستم‌های CI/CD یا ابزارهای دیگر expose شود.
اقدام فوری:
۱. Revoke کردن GOOGLE_CLIENT_SECRET و RESEND_API_KEY
۲. انتقال همه env vars به Vercel Environment Variables
۳. هرگز فایل‌های env را در آرشیو/ایمیل ارسال نکنید
SEC-02 جدید — کشف‌شده
middleware.ts:13
Hardcoded Fallback Secret در Middleware
در middleware.ts یک fallback hardcoded وجود دارد: || 'neuro-it-dev-secret'. اگر NEXTAUTH_SECRET در production تنظیم نشده باشد، مهاجم می‌تواند JWT معتبر با این مقدار شناخته‌شده بسازد و authentication کل سیستم را bypass کند.
اصلاح:
// ❌ فعلی — خطرناک
secret: process.env.NEXTAUTH_SECRET || 'neuro-it-dev-secret'

// ✅ درست — crash کند نه fallback
secret: process.env.NEXTAUTH_SECRET
DB-01 پیوست تأیید شد
prisma/schema.prisma:12
Prisma Schema هنوز SQLite است
علی‌رغم کامنت «Migrated to PostgreSQL» در ابتدای schema.prisma، خط ۱۲ همچنان provider = "sqlite" دارد. در محیط‌های Serverless مانند Vercel، هر cold start تمام داده‌ها را از بین می‌برد.
مهاجرت:
datasource db {'{'}
  provider = "postgresql"
  url = env("DATABASE_URL")
  directUrl = env("DIRECT_URL")
{'}'}

hooks/useCinematicScroll.ts:121

// حذف خط ۱۲۱ — rAF فقط از scroll handler فعال شود

گلوگاه‌های محتوا (Content Clusters)

معماری فعلی فاقد cluster content است. هر صفحه سرویس باید یک hub باشد که محتوای مرتبط را به هم وصل کند:

/* ساختار cluster پیشنهادی برای هر service category */ /services/laptop-services ← hub صفحه ↔ /blog/laptop-screen-repair-cost-london ← blog supporting ↔ /blog/how-to-tell-if-laptop-battery-failing ↔ /areas/camden#laptop-repair ← local page ↔ /areas/islington#laptop-repair ↔ /faq#laptop-questions ← FAQ anchor /* فعلاً این لینک‌ها وجود ندارند — فرصت SEO از دست رفته */
